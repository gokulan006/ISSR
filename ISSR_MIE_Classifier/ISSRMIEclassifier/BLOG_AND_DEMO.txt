Building a Reliable MIE Classifier in a Day: Dual‑RAG, Strict JSON, and a Clean API

Yesterday I took a messy prototype and turned it into a reliable, developer‑friendly MIE (Militarized Interstate Events) classification system. I shipped two Retrieval‑Augmented Generation (RAG) paths, an interactive CLI, a clean Django API, and strict JSON outputs that adhere to research requirements—even when the LLM is offline.

Why this matters
- We need research‑grade structured outputs (MIE action codes, dates, countries, fatalities, etc.).
- It should be easy to run on Windows/macOS/Linux, with pluggable LLM backends (local or cloud).

What I built (quick list)
- Dual RAG:
  - In‑model RAG: fast, in‑memory embeddings to find similar articles.
  - Modular RAG: persistent vectorstore (per year) to search curated datasets.
- Strict JSON output:
  - Layered system prompt, strict schema, robust parsing.
  - Automatic JSON fallback (ML + heuristics) if LLM unavailable.
- API (Django) + CLI:
  - Endpoints to train/query RAG and classify to strict JSON.
  - Quiet CLI with optional JSON printout.
- Data workflow:
  - Per‑year mie_only_YYYY.csv → vectorstores at data/vectorstores/mie/YYYY.

Architecture at a glance
- Enhanced classifier (TF‑IDF + NB + RF) → in‑model RAG → LLM → ensemble decision.
- LLM is pluggable:
  - Local (Ollama) if available.
  - Cloud (Hugging Face Inference API) next; no local GPU needed.
  - If neither available → strict JSON fallback from ML + heuristics.

The two RAG systems (and why both)
- In‑model RAG: Builds sentence embeddings on the training data at runtime. Low latency, great for few‑shot examples.
- Modular RAG: Stored vector DB you can pre‑train and query independently; great for larger/stable contexts and year filtering.

Strict JSON coding output (the key win)
We enforce a schema that includes classification, reasoning, codeable, coding{ micnum, eventnum, ccode1/2, stmon/stday/styear, end*, sidea, action (1..22), fatalmin*/fatalmax*, Location, MeasuringPoint, Purpose (0..4), ForceType (1..3), MinForce/MaxForce }, countries_involved, missing_fields, notes, confrontation_key.
If the LLM returns non‑JSON, we retry once with “JSON only”. If LLM is offline, we return the same schema via ML+heuristics fallback.

Endpoints you can call (Django)
- In‑model RAG + Classifier:
  - POST api/rag/model/init → load data, train ML, build embeddings
  - POST api/rag/model/similar → { text, top_k? }
  - POST api/rag/model/classify_json → strict JSON for { title, subject, text }
- Modular RAG (vectorstore):
  - POST api/rag/modular/train → build a vectorstore
  - POST api/rag/modular/query → semantic search
- Dual‑RAG agent (optional):
  - GET api/agent/catalog/list → build/inspect API catalog index
  - POST api/agent/run → end‑to‑end (API discovery → spec → live data → LLM answer)

CLI that doesn’t shout at you
- python main.py --quiet → minimal, professional prompts and a single verdict line.
- python main.py --quiet --json → prints the strict JSON block.

Data & vectorstores (per‑year workflow)
- Per‑year builds:
  python -m rag.vectorstore.train_from_csv --csv data\raw\mie_only\mie_only_YYYY.csv --out data\vectorstores\mie\YYYY
- Flexible parser supports title/text/probable and Title/Subject /Text/Probable MIE formats.

A quick PRD snapshot
- Goals: strict JSON outputs, minimal setup, swappable LLM backends, two RAG paths.
- Success: ≥99% valid JSON; graceful degradation without LLM; easy endpoints.
- Non‑goals: production auth, fine‑tuning LLMs, full frontend.

Next: switch to Hugging Face Inference API
- Why: zero local model friction, predictable on Windows, fast onboarding.
- Plan:
  - Add LLMProvider interface (env‑driven selector).
  - Implement HFProvider (Authorization: Bearer HF_API_KEY), low temp, token cap, retry on 429/5xx.
  - Keep strict JSON fallback for resilience.

Lessons learned
- Strict JSON needs prompt + parser discipline.
- Dual RAGs are complementary.
- Quiet CLI + clear endpoints make research tools feel production‑ready.


END‑TO‑END EXAMPLE (synthetic)

Scenario
A brief article describes a cross‑border clash. We run it through in‑model RAG and the classifier to get strict JSON.

Input
Title: Border Skirmish Between Country A and Country B Leaves 5 Soldiers Dead
Subject: Military clash in a disputed border region
Text:
“On March 14, 2023, armed forces of Country A crossed into a disputed sector along the Riverdale Crossing. Country B’s border guards engaged, resulting in a short but intense firefight. Country B reported five fatalities among its soldiers. The government of Country A acknowledged the operation as an ‘enforcement action’ against incursions. Country B’s foreign ministry lodged an official diplomatic protest the same day. Ceasefire talks resumed later that evening but ended without agreement.”

How to run
CLI (quiet + strict JSON):
  python main.py --quiet --json
  # Then paste the Title, Subject, Text

API:
  POST http://127.0.0.1:8000/api/rag/model/init
  Body: { "data_path": "data/raw/final_data_true.csv" }

  POST http://127.0.0.1:8000/api/rag/model/classify_json
  Body: {
    "title": "Border Skirmish Between Country A and Country B Leaves 5 Soldiers Dead",
    "subject": "Military clash in a disputed border region",
    "text": "On March 14, 2023, armed forces of Country A crossed into a disputed sector along the Riverdale Crossing..."
  }
